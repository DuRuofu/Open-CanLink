# PC 上位机使用指南

## 安装

```bash
cd software/pc
uv sync
```

## 启动

```bash
uv run python main.py
```

## 标签页说明

### 数据收发

通用串口调试工具。直接发送文本数据，接收显示原始数据。

### CAN 监视器

接收并显示 CAN 总线上的帧。

- **表格列**: 序号、时间戳、CAN ID、帧类型(STD/EXT)、DLC、数据
- **过滤**: 输入 ID（支持十进制或 0x 前缀十六进制）仅显示匹配帧
- **暂停**: 暂停表格更新（数据仍在后台接收）
- **清空**: 清除所有行
- **保存 CSV**: 导出当前表格数据

### CAN 发送

构造并发送 CAN 命令。

- **CAN 控制**:
  - 波特率选择 +「设置波特率」
  - 「CAN Start」/「CAN Stop」启停 CAN 接口
  - 「状态」查询当前总线状态
  - 「设备信息」查询设备固件信息

- **帧发送**:
  - CAN ID（支持 0x 十六进制前缀）
  - 标准帧 / 扩展帧切换
  - DLC 选择（0-8），超出 DLC 的数据字节自动禁用
  - 8 个十六进制数据字节输入
  - 「发送一次」发送单帧
  - 「开始周期发送」按指定周期（ms）重复发送

## 协议

所有通信使用 JSON 协议，详见 [protocol.md](../docs/protocol.md)。

上位机通过 `core/can_protocol.py` 模块处理所有 JSON 编解码：
- `LineBuffer` 处理串口分包
- `parse_message()` 解析设备→主机消息
- `build_*()` 系列函数构建主机→设备命令

## 扩展

### 添加新标签页

1. 在 `ui/tabs/` 创建新类，继承 `QWidget`
2. 实现 `on_data_received(self, data)` 方法接收串口数据
3. 在 `ui/main_window.py` 中导入并注册:
```python
from .tabs.my_tab import MyTab
self.tab_widget.add_tab(MyTab(), "标签名")
```

### 添加新协议命令

在 `core/can_protocol.py` 中添加 `build_xxx()` 函数和对应解析逻辑。
